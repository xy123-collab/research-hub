# 腾讯云一键部署指南（科研数据共享平台）

本指南带你把平台部署到腾讯云。整体形态：**一台 Lighthouse/CVM 跑 Docker（应用 web + nginx 前端）+ TencentDB for PostgreSQL（托管库）+ COS（对象存储）**。你只需开通几项云资源、把密钥填进 `.env`，然后一条命令部署。

---

## 一、需要你提供 / 开通的（照着勾一遍）

下面每一项都对应 `.env` 里的一个配置。带 ★ 的是必须，其余可选。

### 1. 一台服务器（计算）★
- 腾讯云 **轻量应用服务器 Lighthouse**（2C4G 起，最省心）或 **CVM 云服务器**，系统选 **Ubuntu 22.04**。
- 需要给我/你自己：**公网 IP**、SSH 登录方式（用于上传代码、跑部署脚本）。
- 安全组放行端口：**22（SSH）、80（HTTP）、443（HTTPS）**。

### 2. 数据库（TencentDB for PostgreSQL）★
- 控制台创建 **TencentDB for PostgreSQL 15** 实例，**和上面服务器放同一个 VPC/地域**（走内网更快更安全）。
- 你需要提供：数据库 **内网地址、端口(5432)、库名、用户名、密码**。
- 组装成一行填入 `.env` 的 `DATABASE_URL`：
  ```
  DATABASE_URL=postgresql+psycopg2://用户名:密码@内网地址:5432/库名
  ```
- 预算紧张可跳过托管库：在 `.env` 设 `LOCAL_DB=true`，部署脚本会在同机用 Docker 起一个 PostgreSQL（生产仍建议托管）。

### 3. 对象存储（COS）★
用来存数据版本 `.dta`、codebook、附件等。你需要：
- 创建一个 **COS 存储桶**，权限设为**私有读写**（平台用签名 URL 下载，不公开）。
- 提供：**存储桶名**（形如 `research-hub-1300000000`，含 APPID）、**地域**（如 `ap-beijing`）。
- 一对 **API 密钥**：控制台「访问管理 CAM → 密钥管理」新建 **SecretId / SecretKey**（建议用子账号，仅授予该桶的读写权限）。
- 填入 `.env`：`COS_BUCKET / COS_REGION / COS_SECRET_ID / COS_SECRET_KEY`。

### 4. 域名与 HTTPS（可选，但公网正式访问建议做）
- 域名：在腾讯云买/解析一个域名指向服务器公网 IP。**公网对外提供服务需 ICP 备案（预留 2–3 周）**；纯内测可先用 `http://公网IP` 直接访问，跳过备案。
- HTTPS 证书：控制台申请**免费 DV 证书** → 下载 **Nginx 格式** → 把 `.crt/.key` 放到 `deploy/certs/`，参考 `deploy/nginx-ssl.conf.example` 合并进 nginx 配置。

### 5. JWT 密钥★（你本地生成，不用开通任何东西）
- 运行 `openssl rand -hex 32` 生成一串随机值，填入 `.env` 的 `JWT_SECRET`。

### 6. AI（可选，一期可不填）
- 一期 `AI_PROVIDER=none` 即可（勘误 AI 评分、看板 AI 生成会返回“未启用”提示，其余功能不受影响）。
- 二期要用时再提供 `AI_GATEWAY_URL / AI_API_KEY`（走出站脱敏网关，只发元数据，不发原始数据）。

> **一句话清单（复制去准备）**：服务器公网IP+SSH、TencentDB 的连接串、COS 桶名+地域+一对密钥、（可选）域名+SSL证书。JWT 密钥自己 `openssl` 生成。

## 二、部署步骤（三步）

### 步骤 1：把项目传到服务器
在本地项目根目录（含 `backend/ frontend/ deploy/`）执行，或用宝塔/scp 上传：
```bash
# 例：用 rsync 上传整个项目到服务器
rsync -avz --exclude node_modules --exclude '*.db' ./ ubuntu@<公网IP>:/opt/research-hub/
ssh ubuntu@<公网IP>
cd /opt/research-hub
```

### 步骤 2：生成并填写 `.env`
```bash
bash deploy/deploy.sh          # 首次运行会自动从模板生成 .env 并退出
nano .env                      # 按第一节把 ★ 项填好（DATABASE_URL / COS_* / JWT_SECRET）
```

### 步骤 3：一键部署
```bash
bash deploy/deploy.sh
```
脚本会自动：装 Docker（若无）→ 构建前后端镜像 → 起容器 → 后端自动跑数据库迁移(`alembic upgrade head`) + 首次灌入 seed 假数据。

完成后访问：`http://<公网IP>/`，默认账号 `admin/admin123`、`lixiaoyu/pass123`。
健康检查：`curl http://localhost/api/health`。

---

## 三、部署后必做（安全）
1. **改掉默认密码**（admin / lixiaoyu / chenmo 都是 seed 弱密码），或上线前把 `.env` 的 `SEED_ON_START=false` 并清库重灌真实账号。
2. 确认 `JWT_SECRET` 已换成强随机值。
3. 配好 HTTPS（见 1.4），把 `ALLOWED_ORIGINS` 从 `*` 收紧为你的域名。
4. **腾讯云阶段只放合成/脱敏数据**——真实敏感官员数据等第二阶段迁北大校内再导入（详见开发文档 9.2）。COS 开启 SSE 加密、DB 开启加密、全站 HTTPS。

## 四、常用运维命令
```bash
cd /opt/research-hub
docker compose -f deploy/docker-compose.yml logs -f web     # 看后端日志
docker compose -f deploy/docker-compose.yml ps              # 看容器状态
docker compose -f deploy/docker-compose.yml up -d --build   # 改代码后重建上线
docker compose -f deploy/docker-compose.yml down            # 停服务
```

### 备份（建议每日）
```bash
# TencentDB 自带自动备份；应用侧再每日 pg_dump 存 COS：
pg_dump "$DATABASE_URL" | gzip > backup_$(date +%F).sql.gz
# 用 coscmd/cos CLI 上传到 COS
```

## 五、第二阶段：迁北大校内（代码不变，只换配置）
- `DATABASE_URL` 指向校内 PostgreSQL；`STORAGE_BACKEND=local` 或校内对象存储；`AI_GATEWAY_URL` 指向校内 GPU 自托管端点。
- `pg_dump`/`pg_restore` 迁库、文件搬迁、DNS/反代改校内。**业务代码零改动**（storage/ai_client/DB 全走环境变量与接口抽象）。
