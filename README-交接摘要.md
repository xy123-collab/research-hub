# 交接摘要 / README — 科研数据共享平台

> 一页看懂从哪开工。详细规格见《COD数据协作平台-项目开发文档-v1.md》；`prototype/index.html` 是**视觉与流程参考**（纯前端假数据，按钮多为占位）。

## 这是什么
面向**多个课题组**的科研数据共享平台：把各组自建数据变成**可信、可溯源、有版本、贡献可归属**的共享资产，并配学术交流社区。首个课题组 = NSD 发展政经课题组，首个数据集 = COD 地方官员数据库（`officerID`/`termID`）。风格：学术、简约、专业。

## 技术栈（已定）
- 后端 **FastAPI + SQLAlchemy2 + Alembic + Pydantic v2 + JWT(argon2)**
- 前端 **Vue3 + Vite + TS + vue-router + pinia + vue-i18n + Tailwind**（熟 React 可等价替换）
- 数据库 本地开发 SQLite，**生产 PostgreSQL 15**（`DATABASE_URL` 切换）
- 存储 `storage` 接口双实现：本地目录 / **腾讯云 COS**（`STORAGE_BACKEND` 切换）
- AI `ai_client` 接口 + 出站脱敏网关（provider 可换）
- 部署 **Docker Compose**（web/nginx[/db]）

## 部署路线（两阶段，先云后校内）
1. **腾讯云先行**：Lighthouse/CVM + TencentDB PostgreSQL + COS + Nginx/SSL + HTTPS。**仅放合成/脱敏数据**做内测。
2. **迁北大校内**：换 `DATABASE_URL` + `storage` 到校内，导入真实敏感数据；代码不变。
> ⚠️ 敏感官员数据默认不上商业云；腾讯云阶段用假数据。

## 核心概念（务必先理解，别做成又一个飞书）
- 价值核心 = **数据可信化 + 贡献可归属兑现**，不是"更好用的聊天/看板"。
- **三级权限**（角色在创建时产生、可交接）：
  - `super_admin` 总管理员＝**平台维护者**：只管账号/运维/元信息，**看不到课题组/数据内部内容与贡献，不能单方面删数据**（隐私隔离）。
  - `group_admin` 课题组管理员：管本组成员/公约/贡献总览。
  - dataset `founder`/maintainer/member：管数据集内容与授权。
- **数据集非成员**：可看 codebook + 数据看板，**不能下载原始数据**。
- **下载分级**：成员下当前版，管理员下全部历史。
- **勘误＝评分制审核**：提交 →（成员评分 0–10 + AI 评分）→ 管理员终审(采纳程度+最终分)→ 发版修复 → 贡献**按终审分加权**。
- **数据核验**：规则+AI 只**发现与起草**，**绝不静默改原始数据**；采纳走上面的勘误管线，人工只剩"发版时一次批量确认"。
- **版本不可覆盖**；版本号 `v1.0.1`/`-hotfix`/`v1.1.0`/`v2.0.0`。
- **公约**：进入课题组/数据集页面时弹出确认（非登录）；对应管理员可编辑（版本化）。

## 从哪开工（顺序）
1. **M0** 脚手架 + `/deploy`（Docker/nginx/`.env.example`）+ `storage`/`ai_client` 接口 → **尽早在腾讯云拉起最小可运行版**。
2. **M1** 按文档第五章建全部表 + Alembic 迁移 + seed（总管理员、NSD 组、2–3 数据集含发起人+必填联系方式、公约、权限码、变量/版本/代码/帖子/贡献）；审计中间件 + `record_contribution()`。
3. **M1.5** 三级权限判定（super→group→dataset）+ 公约确认门禁 + 非成员下载/写入拦截。
4. **M2** 多课题组/数据集/双勘误评分审核全链路 + **M2.5 数据核验专栏**（flag→勘误草稿，不改数据）。
5. **M3** 社区（发帖可见性三级+评论/点赞完整 CRUD）+ **项目工作台完整 CRUD**（进展/待办/讨论/文件/Overleaf）+ **Skill 共享** + **个人简历** + 代码"一键生成数据处理说明"。
6. **M4** 看板（样本描述统计+自定义维度分布+**只读沙箱** AI/手写分析）+ 文献地图。
7. **M5** 定时质量巡检 + AI 网关脱敏。
8. **M6** 三级管理后台 + 测试 + 验收。
