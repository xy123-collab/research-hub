## 三、一键更新部署（最常用）

在项目文件夹里，改完文件后执行三条命令：

```bash
git add -A
git commit -m "描述这次改了什么"
git push
```

push 完成后 **Render 会自动重新构建并上线**（几分钟）。到 Render 面板 → 服务 `research-hub` → **Events** 标签，看到 **"Deploy live"** 就代表新版本已生效。**这就是"一键更新"——你只需要 push。**

> 改环境变量（如换 AI key、切换存储）不用 push：Render 面板 → 服务 → **Environment** → 改 → Save，会自动重部署。

## 四、两个特殊情况

**1. 改了数据库表结构**（在 `models/*.py` 里加/改字段）

分两种情况：

- **新增一张表**（只加新 model，不动老表）：**不用写迁移**。`seed.py` 启动时会 `Base.metadata.create_all()`，只会补建"不存在的表"，Render 上线时自动创建。本轮新增的 `dataset_group_requests` 表就是这么上的。
- **改老表的字段**（加列/改类型/改约束）：`create_all` **不会** ALTER 已存在的表，必须写迁移，否则线上库不同步：
  ```bash
  cd backend
  alembic revision --autogenerate -m "描述改动"   # 生成迁移
  git add -A && git commit -m "db: 改动" && git push
  ```
  容器启动会自动 `alembic upgrade head` 应用迁移。

> 注意：`SEED_ON_START=true` 时 create_all 才会跑（默认开）。生产上若关了 seed，新增表也要走 alembic。

**2. 本地先跑起来测试（可选，推荐）**
```bash
# 后端
cd backend && pip install -r requirements.txt
export DATABASE_URL="sqlite:////tmp/dev.db"
python -m app.seed && uvicorn app.main:app --reload    # http://localhost:8000/docs
# 前端
cd frontend && npm install && npm run dev               # http://localhost:5173
# 跑测试
cd backend && python -m pytest tests/ -q
```
本地验证没问题，再 push 上线，最稳。

## 六·补充3（数据集精修：数据分类 / 脱敏 / 批量勘误自动改数 / 代码库版本）

> 本轮给后端加了 `pandas`、`openpyxl` 依赖（见 `requirements.txt`），用于读写 `.dta` 和生成 Excel 模板。**免费档内存吃紧**，所有数据处理都做了「服务器直接处理，超 40MB 或设为仅脚本模式则回退成脚本」的双路策略（`services/data_ops.py`）。pandas 在函数内惰性导入，不拉高常驻内存。

**新增表**（新表，`create_all` 自动建，见 `models/curation.py`）：`version_extra`(版本数据分类) `dataset_data_config`(唯一ID/仅脚本) `variable_mask_rules`(脱敏规则) `bug_items`(批量勘误子项) `code_versions` `code_grants` `code_comments`。

**版本数据分类**（`api/datasets.py`）：发版新增 `data_kind`(raw|masked|sample)。样例=公开可下、独立不迭代；原始+脱敏同步竞争 `is_current`。`GET .../versions` 返回 `data_kind/has_data/masked_source`。样例数据下载对所有登录用户开放（`check_download` 里判断）。

**一键脱敏**：`PUT /datasets/{slug}/data-config`（唯一ID变量 + 每变量脱敏动作 keep/drop/hash/bucket + `script_only`）；`POST /datasets/{slug}/versions/{vid}/desensitize` 从原始版生成脱敏版（服务器读 .dta 按规则处理→写新版；回退时返回 Stata 脚本文本）。

**批量勘误 + 自动改数**（`api/bugs.py`）：
- `GET /datasets/{slug}/bug-template` 下载 Excel 模板（含列头批注、变量清单、填写说明）。
- `POST /datasets/{slug}/bugs/batch` 上传 Excel/CSV → 集成为**一条 Bug + 多个 BugItem**。单条提交也会建一个 BugItem，统一按子项处理。
- 逐条终审：`POST /bug-items/{iid}/ai-review`、`POST /bug-items/{iid}/finalize`（自审校验同权限规则）。父 Bug 状态按子项聚合。
- `POST /datasets/{slug}/apply-corrections`：把「已采纳未应用」的 BugItem 按 `唯一ID+变量名` 定位，自动改基准版 .dta → 生成新原始版本，并把子项标 fixed；超限回退脚本。**底层不写死变量名**，靠唯一ID变量+变量名规则定位，换数据库也适用。

**代码库**（`api/code.py`）：`code_versions` 版本迭代（发布必须写 `changelog`）；`code_grants` 作者授予他人「可修改/可重发布」；成员 `GET /code/{cid}/download` 下载代码文件；`GET/POST /code/{cid}/comments`（`is_correction` 标记勘误类评论）。`GET /code/{cid}` 现返回 versions/grants/can_edit/can_publish/can_grant。

**看板 AI 修正**：`gen_analysis` 的提示词现在明确写清沙箱里 `df` 的真实四列（`var_name/group_key/bucket/value`，即派生汇总长表）+ 从 `Variable` 表自动读出的变量清单与分组，AI 不再用不存在的原始列名。沙箱本身：后端 `multiprocessing` 子进程、静态过滤禁写/禁网、受限 builtins、10s 超时（`services/sandbox.py`）。

**前端**：`Dataset.vue` 加悬浮「下载最新数据」按钮、版本库数据分类标签/生成脱敏/一键应用勘误/数据处理设置、批量勘误模板下载与导入、勘误详情逐条子项打分、代码弹窗版本/权限/下载/评论、成员默认显示3个+检索弹窗、看板说明；`Group.vue` 公约移到成员上方、成员检索弹窗。

**测试**：`tests/test_curation.py` 覆盖数据配置/唯一ID、模板下载、批量集成+逐条终审、应用勘误写新版、样例公开下载、脱敏生成、代码版本权限与评论。连同权限用例共 **30 个单测全通过**。
