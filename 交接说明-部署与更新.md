# 交接说明 · 部署现状与更新流程

> 给下一个协作者：平台已上线，代码在 GitHub，push 到 `main` 即自动部署到 Render。
> 本文只讲**现状 + 怎么改 + 关键坑**；功能细节见 `平台规则/`（01–12）与 `产品功能/`。

## 一、部署现状

- **仓库**：`https://github.com/xy123-collab/research-hub`，主分支 `main`（前端/后端/部署配置都在这）。
- **Render**：`render.yaml` 蓝图部署。一个 Web 服务 `research-hub`（Docker，FastAPI 同源托管前端静态资源）+ 一个免费 PostgreSQL（自动注入 `DATABASE_URL`）。线上 `https://research-hub-pmow.onrender.com`。**`autoDeploy` 开启 = push 即上线。**
- **AI**：腾讯云 TokenHub（OpenAI 兼容），`AI_API_KEY` 在 Render Environment 配，不写进代码。
- **配置/密钥**都在 Render 面板 Environment（改这些不用 push，Save 即自动重部署）。

**免费档限制（重要）**：
- 闲置 15 分钟休眠，冷启动约 1 分钟（第一次访问很慢，非 bug）。
- 免费 PG 约 30 天到期，到期前需导出/升级/换 Neon。
- **无持久磁盘：上传的文件（.dta/codebook 等）重启即清空**（账号/勘误/贡献/帖子等在 PG 里不丢）。要长期存文件→切腾讯云 COS（见第八节，只改环境变量）。
- **512MB 内存：`WEB_CONCURRENCY` 必须保持 1**（多 worker + pandas 沙箱会 OOM）。用户变多就升付费实例（约 $7/月，不休眠），再调回 2。
- **出站 SMTP 端口（25/465/587）被封**（2025-09 起）→ 免费档发不了邮件，见第七节。

## 三、一键更新

改完文件在项目根目录执行：

```bash
git add -A && git commit -m "描述改动" && git push
```

push 完 Render 自动构建上线（几分钟）；面板 → 服务 → Events 看到 "Deploy live" 即生效。

## 四、改数据库表结构

- **新增一张表**（只加新 model、不动老表）：**不用写迁移**。`create_all()` 启动时自动补建不存在的表。绝大多数新功能都走这条。
- **给老表加列/改约束**：`create_all` 不会 ALTER，**必须写 alembic 迁移**，否则线上库不同步：
  ```bash
  cd backend && alembic revision --autogenerate -m "描述"   # 或手写，参考现有迁移
  ```
  容器启动自动 `alembic upgrade head`。迁移都加了「表不存在则跳过」守卫，全新库先 create_all 也不报错。
- **当前迁移链（head = `a5b6c7d8e9fa`）**：`a1b2…`(post_comments.parent_id/created_at)→`a2b3…`(lit_refs.doi)→`a3b4…`(user_profiles.email)→`a4b5…`(posts.title/post_type/status/created_at + post_reactions.created_at)→`a5b6…`(user_profiles.email_opt_in)。

**本地跑起来测试**（推荐先本地验证再 push）：
```bash
cd backend && pip install -r requirements.txt
export DATABASE_URL="sqlite:////tmp/dev.db"
python -m app.seed && uvicorn app.main:app --reload      # localhost:8000/docs
python -m pytest tests/ -q                               # 68 个单测
cd ../frontend && npm install && npm run dev             # localhost:5173
```

## 五、当前功能全景

平台以**数据集为中心**，课题组为可选容器（`datasets.group_id` 可空）。五大分区 + 个人入口：

- **数据集**（首页 `Home.vue` / 详情 `Dataset.vue`）：版本库（数据分类 raw/masked/sample、一键脱敏、对照表/codebook）、勘误（批量 Excel + 逐条终审 + 自动改数）、处理代码库（版本/授权/评论）、数据看板（沙箱接真实 .dta 只读分析）、文献地图（AI 综述 + 词云 + 真实性核验）、研究讨论区、成员与权限、AI 勘误助手（左下角悬浮）。
- **课题组**（`Groups.vue`/`Group.vue`）：组内动态（每条可跳转）+ 组内讨论。
- **研究讨论区 = 统一帖子系统**（`Feed.vue` + `PostCard`/`PostComposer`）：研究广场三栏（热榜/信息流/筛选，筛选写 URL）、数据集讨论区、课题组讨论、个人主页"我的讨论"**共用同一份帖子**，改动即时同步。帖子有类型/标题/标签/可见范围/附件/关注/状态；评论支持回复+点赞，评论数统一口径（99+/千+）。详见 `平台规则/11`。
- **其他协作**（`Collab.vue`）：Skill 协作 + 用户自建协作分区。
- **个人主页**（`Profile.vue`）：编辑资料（头像/研究方向/关键词/公开邮箱/注册邮箱/邮件提醒开关）、整块文本简历（+AI 一键排版）、在做项目（标题/正文/图片/标签/可见范围/置顶/评论/删除）、我的讨论、我的贡献、项目工作台（时间轴 + 拉人踢人）。
- **权限**：两级管理员（总管理员/普通管理员，见 `平台规则/01`）；加入≠下载权，单独授权走 `DatasetGrant`；**可见范围四类**（全平台/课题组/数据集/仅自己）统一走 `content_scopes` + `scope_visible()`，**总管理员不绕过**（仅自己可见对管理员也生效，见 `平台规则/06`）。
- **贡献度**：创建数据集/提交代码/文献上传（按次）/勘误采纳（按条）/发帖等计分；本人看全明细、他人看公开汇总，见 `平台规则/12`。
- **消息中心**（`NotificationCenter.vue`）：右下角悬浮铃铛 + 红色数字角标（未读数），分类聚合待办与互动通知，60 秒轮询，「全部已读」游标存 `notification_state`。
- **邮件**：注册找回密码、每日 8:00/18:00 摘要（`digest.py`，尊重邮件提醒开关）；抽象层 `email_service.py` 可插拔（见第七节）。
- **账号**：注册必填邮箱；找回密码 `ResetPassword.vue`；名称去重 `core/naming.py`（用户名/邮箱/slug/name_zh/版本号/变量名唯一）。

规则文档 `平台规则/00–12`（总览/权限/数据集/版本/下载/勘误/可见范围/消息邮件/账号安全/沙箱/去重/讨论区/贡献度）是**权威规格**，改逻辑前先看对应号。

## 六、邮件配置（腾讯企业邮 SMTP）

代码已支持 SMTP（`EMAIL_BACKEND=mock` 时只记录不发；设 `smtp` 即真发）。在 Render Environment 配：

| 变量 | 值 |
|---|---|
| `EMAIL_BACKEND` | `smtp` |
| `SMTP_HOST` | `smtp.exmail.qq.com` |
| `SMTP_PORT` | `465`（隐式 SSL，代码自动识别）或 `587`（STARTTLS，需 `SMTP_TLS=true`） |
| `SMTP_USER` / `EMAIL_FROM` | 完整邮箱地址（发件人须=认证账号） |
| `SMTP_PASSWORD` | 邮箱「客户端专用密码」（不是登录密码） |

⚠️ **Render 免费档封了出站 SMTP 端口**，所以免费档上会报 "Network is unreachable"，无论 465/587 都发不出。要真正发信：**升级 Render 付费实例**，或**部署到校内服务器**（本平台本就为校内部署设计，`docker-compose.yml`），或换 HTTP 邮件 API（SendGrid/腾讯云 SES，走 443）。代码本身没问题，465 隐式 SSL 已支持。

用户可在「编辑资料」里开/关**邮件提醒**（`user_profiles.email_opt_in`，注册默认开，开关都发确认信，摘要尊重开关）；也能查看/修改注册邮箱（`/me/account-email`，唯一+格式校验）。

## 七、关键坑与约定（务必看）

- **建表策略**：新功能一律用新表（`create_all` 免迁移）；只有给老表加列才写 alembic。
- **鉴权下载**：需登录的下载接口（模板/导出/附件/skill/代码/版本文件等）**不能用 `window.open`**（不带 token→401），统一用 `utils/download.ts` 的 `downloadFile(url, name)`。
- **删除有子表的记录**：本会话 `autoflush=False`，删父行前**必须按「先子后父」用立即批量删除**清子表（`db.query(...).delete()`），不要混用 ORM `db.delete()`，否则提交时 flush 顺序不定、Postgres 外键报错（SQLite 不报，易漏测）。参考 `posts.py` 的 `del_project`/`del_post`。
- **可见范围**：`scope_visible()` 不给总管理员开后门；给帖子/项目加接口时记得按可见范围鉴权（评论、点赞、附件都要）。
- **AI 依赖**：AI 综述/勘误助手/文献核验/引文匹配都依赖 `AI_API_KEY`，未配时返回"AI 未启用"占位、**不阻断**主流程；所有 AI 调用**只送元数据/标题，绝不上传原始数据行**。
- **data_fixes**（`app/data_fixes.py`）：一次性/幂等数据修正靠 `applied_fixes` 表标记。要再改某账号/清某数据集，**换一个新 marker key**（如 `xxx_v2`），否则旧标记存在会跳过。涉及账号的修正只在"找到并成功更新"后才写标记（账号没注册就下次重试）。
- **git 锁**：沙箱工具提交后 `.git` 可能残留锁文件，push 前先 `rm -f .git/index.lock .git/HEAD.lock .git/refs/heads/main.lock`。
- **测试**：后端 `cd backend && pytest tests/ -q`（68 个）；前端 `cd frontend && npx vite build --outDir /tmp/x --emptyOutDir`（只校验编译，不动 dist）。

## 八、切腾讯云 COS（解决"重启丢文件"，只改环境变量）

代码已内置 COS 后端（`core/storage.py`，依赖已在 requirements）。**切换不用改代码、不用 push**。

1. **建桶**（COS 控制台）：名 `<名字>-<APPID>`（如 `research-1445229278`，不可改）；地域就近（如 `ap-beijing`）；单 AZ；**私有读写**（绝不公有读）；版本控制关。
2. **建密钥**（CAM → 子用户）：勾"编程访问"，授 `QcloudCOSFullAccess`；**立即保存 SecretId/SecretKey**（Key 只显一次）。
3. **Render Environment** 设 5 个变量后 Save：`STORAGE_BACKEND=cos`、`COS_BUCKET`、`COS_REGION`、`COS_SECRET_ID`、`COS_SECRET_KEY`。
4. **验证**：发一版数据后，COS "文件列表"里出现 `versions/...` 即接通；此后重启不丢文件。切回本地存储把 `STORAGE_BACKEND` 改回 `local`。
