**常见改动去哪改：**

| 要改什么 | 改哪个文件 |
|---|---|
| 页面/界面/交互 | `frontend/src/views/*.vue`、`components/*.vue` |

| 存储/AI/密钥等配置 | Render 面板「Environment」（或本地 `.env`） |

## 三、一键更新部署（最常用）

在项目文件夹里，改完文件后执行三条命令：

```bash
git add -A
git commit -m "描述这次改了什么"
git push
```

push 完成后 **Render 会自动重新构建并上线**（几分钟）。到 Render 面板 → 服务 `research-hub` → **Events** 标签，看到 **"Deploy live"** 就代表新版本已生效。**这就是"一键更新"——你只需要 push。**

> 改环境变量（如换 AI key、切换存储）不用 push：Render 面板 → 服务 → **Environment** → 改 → Save，会自动重部署。

## 四、两个特殊情况

**1. 改了数据库表结构**（在 `models/*.py` 里加/改字段）

分两种情况：

- **新增一张表**（只加新 model，不动老表）：**不用写迁移**。`seed.py` 启动时会 `Base.metadata.create_all()`，只会补建"不存在的表"，Render 上线时自动创建。本轮新增的 `dataset_group_requests` 表就是这么上的。
- **改老表的字段**（加列/改类型/改约束）：`create_all` **不会** ALTER 已存在的表，必须写迁移，否则线上库不同步：
  ```bash
  cd backend
  alembic revision --autogenerate -m "描述改动"   # 生成迁移
  git add -A && git commit -m "db: 改动" && git push
  ```
  容器启动会自动 `alembic upgrade head` 应用迁移。

> 注意：`SEED_ON_START=true` 时 create_all 才会跑（默认开）。生产上若关了 seed，新增表也要走 alembic。

**2. 本地先跑起来测试（可选，推荐）**
```bash
# 后端
cd backend && pip install -r requirements.txt
export DATABASE_URL="sqlite:////tmp/dev.db"
python -m app.seed && uvicorn app.main:app --reload    # http://localhost:8000/docs
# 前端
cd frontend && npm install && npm run dev               # http://localhost:5173
# 跑测试
cd backend && python -m pytest tests/ -q
```
本地验证没问题，再 push 上线，最稳。

## 六、本轮（数据集优先重构 + 视觉精修）改了什么

给下一个线程快速对齐"现在长这样"：

**导航与首页**
- 顶部导航：**数据集(首页) · 课题组 · 研究广场 · 我的主页 · Skill共享 · 管理后台**。首页从"先选课题组"改为**数据集优先**：进来直接是数据集网格（我参与的 + 发现），课题组挪到独立的 `Groups.vue` 列表页（`/groups`）。
- 首页顶部有"价值 hero + 四个协作能力卡"（勘误 / 只读数据看板 / 规则+AI核验 / 处理代码库），并有"近期更新"条（汇总我的数据集的版本/勘误消息，点一条直达数据集）。
- 搜索支持**按名称或 ID**匹配数据集；课题组列表页也有同样搜索。

**数据集归属（重要，权限逻辑）**
- 数据集可**独立创建**（不属于任何课题组），`datasets.group_id` 允许为空。创建时可选填"归属课题组（按名称或 ID，仅限我已加入的组）"，留空即独立。
- **已创建的数据集不允许再进出课题组**（本轮为简化逻辑砍掉了"申请并入/移出"的前端入口）。归属在创建时一次定死。
- 后端仍保留了 `attach-request/detach-request` 接口和 `dataset_group_requests` 表（当前前端未用）。若下个线程要恢复"事后调整归属+课题组管理员审批"，后端现成的，接回 UI 即可（见 `api/groups.py` 的 `dataset-requests` / `dataset-group-requests/{id}/decide`）。

**数据集详情页**
- 新增两个页签：**更新记录**（版本/勘误/代码时间线，带下拉筛选，接口 `/datasets/{slug}/activity`）、**研究广场**（该数据集 tag 的讨论 + 成员发帖，`/posts?dataset_id=`）。
- 顶部有成员协作快捷条（提交勘误/代码库/核验/发版）。

**课题组详情页**
- 新增"组内动态"（成员发帖 + 数据更新，接口 `/groups/{slug}/activity`）。

**视觉**
- 全站 slogan 改为"**让自建数据可信、可迭代、可复用**"（i18n + `Login.vue` + 后端 `config.py` 的 `PLATFORM_SLOGAN_*` 都改了）。
- emoji 统一换成线性图标组件 `components/Icon.vue`。
- 数据集与课题组都展示**唯一 ID**（用数据库主键），各处检索支持 ID 或名称。

**新增只读接口**（`api/datasets.py`）：`/datasets/mine`（我参与的，带活跃度信号+近期消息）、`/datasets/{slug}/activity`；`api/groups.py`：`/groups/{slug}/activity`。`POST /datasets` = 独立建集。

## 六·补充（本轮：按《权限规则》全面对齐 + 消息中心）

对照 `讨论/ly_科研数据协作平台权限规则.docx` 重构了底层权限并补齐操作按钮：

**权限内核（`backend/app/core/permissions.py`）**
- 严格落实「原则一」：平台总管理员不再自动拥有课题组/数据集内容权限（`is_super_admin` 只用于平台系统路由，不再短路内容判定）。总管理员若要看内容须按普通流程加入。
- 支持**多管理员**与增/删/交接：数据集 `ds_role in (founder|admin)`、课题组 `group_role=group_admin`，删除/降级时**至少保留一名管理员**。
- 「原则三」加入数据集≠下载权：审批加入不再自动发下载权限；单独授权走新表 `DatasetGrant`（支持永久/指定日期/指定版本/指定项目到期规则）。

**新增表（都是新表，靠 `create_all` 自动建，无需迁移）**：`dataset_grants` `dataset_settings` `download_requests` `version_candidates` `codebook_drafts`（见 `backend/app/models/access.py`）。

**新增/改动接口**（`api/datasets.py`、`api/groups.py`、`api/bugs.py`）：
- 下载分级与策略：数据集设置 `download_policy`(public/member/approval/masked_only/forbidden)；下载申请→审批（`/datasets/{slug}/download-requests`、`/download-requests/{id}/decide`）；历史版本单独授权。
- 单独授权：`/datasets/{slug}/members/{uid}/grant`（带到期）、`/revoke`。
- 管理员管理：数据集 `/datasets/{slug}/admins/{uid}`（POST 增 / DELETE 删）；课题组 `/groups/{slug}/admins/{uid}`、`/groups/{slug}/members/{uid}`、`/groups/{slug}/join-requests`。
- 在线分析、上传候选文件（`/candidates`）、codebook 草稿（`/codebook-draft`）均需对应授权。
- 数据集设置/关闭：`/datasets/{slug}/settings`、`/datasets/{slug}/close`。
- 勘误自审校验：管理员本人提交的勘误，若还有其他管理员则须由另一人终审（仅一名时允许并写审计 `self_review`）。

**消息中心（悬浮入口）**：后端 `api/notifications.py` 的 `GET /notifications` 按当前用户角色实时聚合待办（加入审批/下载审批/勘误终审/归属审批）与「与我相关」的状态变更；前端 `components/NotificationCenter.vue` 是右下角悬浮铃铛+红点角标+面板，挂在 `App.vue`，每 60 秒轮询，点条目直达对应页面。

**前端补齐按钮**：`Dataset.vue` 新增「成员与权限」页签（加入/下载审批、授权与撤销、增删管理员、成员移除）、数据集设置弹窗、下载申请弹窗、上传候选文件弹窗，并按权限显隐下载/在线分析按钮；`Group.vue` 增加加入审批、成员管理、编辑课题组；`Admin.vue` 增加按 ID 添加/交接总管理员。

**测试**：`tests/test_permission_rules.py` 覆盖总管理员边界、审批下载全链路、在线分析授权、唯一管理员保护、勘误自审拦截、消息中心。全部 21 个单测通过。
