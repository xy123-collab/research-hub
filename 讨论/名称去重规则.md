# 名称去重规则（#1）

规则集中实现在 `backend/app/core/naming.py`（`normalize_name` + `ensure_unique`），
创建与重命名接口统一调用。比较口径：**去首尾空格、连续空白压成一个、大小写不敏感**（casefold）；
库里仍保存用户原样输入，只在判重时归一化。

## 一、必须唯一（提交前自动检索，重复则拦截并提示「已存在」）

| 名称 | 范围 | 落地位置 |
|---|---|---|
| 账号用户名 `username` | 全局 | `api/auth.py` 注册（原有） |
| 注册邮箱 `email` | 全局 | `api/auth.py` 注册（原有） |
| 数据集机器名 `slug` | 全局 | 创建数据集（原有） |
| **数据集名称 `name_zh`** | 全局（未软删的） | 独立建集 / 组内建集 / 重命名，本轮新增 |
| 课题组机器名 `slug` | 全局 | 创建课题组（原有） |
| **课题组名称 `name_zh`** | 全局（未软删的） | 创建 / 编辑课题组，本轮新增 |
| 版本号 `version_id` | 同一数据集内 | 发版（原有，「不可覆盖」） |
| 变量名 `var_name` | 同一数据集内 | 由 .dta 抽取，天然唯一 |

重命名时排除自身（按主键 id），改回原名或大小写微调不会误报。

## 二、允许重复（不拦截）

- **用户显示名 `display_name`**：同名同姓正常，唯一键是用户名 / ID。
- **文献显示名 / 标题 `title`**：不同数据集常引同一篇，允许重复（符合你的要求「文献上传时显示名可以重复」）。
- 帖子标题、评论、工作台条目、公约、changelog、对照表说明等自由文本。
- 处理代码 / Skill 的标题：版本化资源，靠 ID 区分，允许重名。

## 三、前端提示

创建 / 重命名接口返回 400 + 明确中文（如「数据集名称「X」已存在，请换一个」）。
`Home.vue`、`Groups.vue` 的创建流程都会 `alert(detail)`，用户即时看到冲突原因。
