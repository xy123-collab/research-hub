## 二、核心闭环现在能完整跑通 ✅
下载数据 → 发现问题 → 提交 bug + **上传证据附件** → 成员评分 + AI 评分 → **管理员点开 bug 详情终审** → 发布新版本时**勾选本次修复的已采纳勘误**（自动标 fixed + 记录修复版本号）→ 旧版本文件保留、下载分级（成员下当前版、管理员下全部历史）。这条链已有单测覆盖（`test_full_core_loop_...`）。

## 三、AI 功能（已接你的腾讯云 TokenHub）
默认接 `https://tokenhub.tencentmaas.com/v1`（OpenAI 兼容）。第一版启用这几项——都**只发元数据/公开文本，绝不发原始数据行**，安全：
- **AI 勘误评分**：bug 详情里「触发 AI 评分」，AI 给 0–10 可采纳度分，辅助管理员终审。
- **AI 生成分析代码**：看板里输入自然语言 → AI 生成 pandas 代码 → **只读沙箱**执行（禁写回/禁网络）。
- **AI 数据处理说明**：代码详情「一键生成数据处理说明」，出论文数据附录写法草稿。
- **AI 文献/用途总结**：文献页「AI 总结研究用途/话题」。

暂不建议第一版做的：直接让 AI 读原始数据做核验（应始终"只发现、走勘误管线"，现有规则/AI flag→草稿的设计已经够用）。
模型：默认快模型 `deepseek-v4-flash`（评分/总结）、强模型 `deepseek-v4-pro`（代码/写作），可在环境变量改。

> ⚠️ **安全提醒**：你的 API key 已经在对话里明文出现，等于泄露，请到腾讯云**重新生成一把新 key**。key **只填在 Render 环境变量里**，我没有写进任何代码或提交。

# 如何更新到网址（关键）
Render 已开 `autoDeploy`，**你只要 push 到 GitHub，Render 会自动重新构建上线**。在项目目录里：

```bash
git add -A
git commit -m "优化：上传/CRUD/AI/核心闭环"
git push
```

推完等 Render 自动构建完成（几分钟）即生效。

## 还要做一次：在 Render 配置 AI key（只需一次）
1. Render 面板 → 你的服务 **research-hub** → **Environment**。
2. 找到 `AI_API_KEY` → 填入你**新生成**的 TokenHub key → Save（会自动重部署）。
3. `AI_PROVIDER` 已设为 `tokenhub`；想临时关掉 AI 就改成 `none`。

> 说明：`render.yaml` 里 `AI_API_KEY` 标了 `sync: false`，意味着它不写进代码仓库，必须在面板手填——这是防泄露的正确做法。

## 数据会不会丢？
- 账号、勘误、贡献、帖子等都在 PostgreSQL，重部署**不丢**。
- 上传的文件（免费档无持久盘）在重启后会清空——**演示够用**；要长期保留就把 `STORAGE_BACKEND` 换成腾讯云 COS（填 COS_* 环境变量），或迁北大校内存储。
