# 科研数据共享平台 · 产品功能介绍

> Research Hub —— 让每一份自建数据都可信、可复用、可归属
> 北京大学国家发展研究院 · 智慧科研团队

---

## 一、这是什么

科研数据共享平台是一个面向**多个课题组**的开放科研数据协作空间。它要解决的不是"再做一个更好用的聊天或看板"，而是一个科研场景里更根本的问题：课题组里那些分散的、私有黑箱的、被反复重建的自建数据，如何变成一批**可审计、可溯源、有质量保证、贡献可归属**的共享研究资产；并在其上长出一个低成本的**学术交流社区**，让"谁围绕这些数据做了什么、想了什么"被沉淀和承认下来。

平台首个课题组是 **NSD 发展政经课题组**，首个数据集是 **COD 地方官员数据库**（核心识别变量 `officerID`、`termID`）。但平台本身是通用的——任何课题组都能入驻、发起自己的数据集与协作。

产品由三个面构成，贯穿始终：**数据面**（版本、下载、勘误、代码、看板）、**社区面**（研究广场 + 个人主页）、**治理面**（三级权限、公约、贡献总览、质量巡检）。三者由一条主线串起来：**贡献记录 → 多通道兑现**。

## 二、信息架构：平台长什么样、功能在哪

顶部导航固定四个入口：**首页 · 研究广场 · 我的主页 · 管理后台**，右上角是中英语言切换与账号。整体结构如下：

```
登录/注册（登录即"你本人"，不选身份；进入课题组/数据集时弹公约确认）
└─ 首页 Home
   ├─ 我的课题组（卡片：成员数/数据集数/我的角色）＋［创建课题组］
   ├─ 发现课题组（可申请加入 → 课题组管理员审批）
   └─ 研究广场·近期（最新想法流）
   │
   ├─▶ 课题组详情 Group：公约栏 · 数据集网格 ＋［发起新数据集］· 成员
   │
   ├─▶ 数据集详情 Dataset（7 个页签）
   │     概览 · 数据看板 · 版本库 · 原始勘误 · 处理代码库 · 文献地图 · 数据核验
   │
   ├─▶ 研究广场 Feed：发帖/想法流，点赞·评论·标签·可见性三级
   ├─▶ 我的主页 Profile：贡献度 banner ＋ 在做的项目 / 我的贡献 / 个人简历 / 项目工作台
   ├─▶ Skill 共享：科研技能包（与数据集同构）
   └─▶ 管理后台 Admin：按持有角色展示（总管理员 / 课题组管理员 / 数据集管理员）
```

设计基调是"学术、简约、专业"：墨色文字、暖白背景、单一深靛蓝强调色、三线表、克制的卡片流，社区借用小红书式低门槛发帖但视觉保持学术冷静感。全站中英双语即时切换。

## 三、角色与权限：三级体系，角色在创建时产生、可交接

平台最核心的治理设计是：**身份不在登录时选择**——登录即"你本人"。你持有哪些角色，是在"创建课题组 / 发起数据集"时**自动产生**的，并且**可以交接**给他人。权限判断按 `总管理员 → 课题组 → 数据集` 逐级下沉。

**站点级**

| 角色 | 是谁 | 能做什么 |
|---|---|---|
| 总管理员 / 平台维护者（可多个、可交接） | 平台运维 + 导师 | 只管**平台运维层**：账号启停/重置密码、课题组元信息审批、平台配置与 AI 网关、存储备份、动作元数据级审计 |
| 普通成员（含新注册空白账户） | 每个用户 | 发帖、创建课题组、申请加入、编辑个人主页；未加入任何组也能在广场/主页活动 |

> **总管理员的隐私边界（重要设计）**：总管理员是平台维护者，不是内容超级用户。它**默认看不到任何课题组/数据集的内部内容、原始数据、成员贡献明细或研究想法**（只见名称/负责人/成员数等元信息）；审计日志对它只暴露"谁在何时做了什么类型的操作"，不暴露内容与被改的具体值；它**不能单方面删除数据**（删除须由所有者发起或走合规流程，且一律为可审计的软删除）。这条边界是为了让敏感科研数据在平台上也有隐私隔离。

**课题组级**（作用域限单个课题组）

| 角色 | 能做什么 |
|---|---|
| 课题组管理员（可多个、可交接） | 审批加入本组申请、管理成员、编辑课题组公约、在本组下管数据集、查看**本组全员总贡献度 + 分数据集贡献度** |
| 课题组成员 | 浏览本组数据集与想法、在本组发帖、申请加入本组某数据集处理 |

**数据集级**（作用域限单个数据集）

| 角色 | 能做什么 |
|---|---|
| 发起人 / 数据集管理员（可交接） | 该数据集内最大权限：发版、勘误**终审**、管理成员与授权、编辑公约与文献、审批加入申请；可给成员授权，最高可授到与自己一致 |
| 维护者 maintainer | 勘误写入、跑质量巡检、维护代码库 |
| 成员 member | 按授权分级：默认可下载数据、提交/评审勘误 |
| **非成员**（任意登录用户） | **可看 codebook + 数据看板，但不能下载原始数据、不能提交勘误**；要参与须"申请加入处理"经审批 |

## 四、功能分块详解（含位置与权限）

### 4.1 登录 / 注册
**位置**：入口页。登录即本人，不选身份。首页与登录页底部有统一品牌页脚（平台名 + slogan + 北大国发院智慧科研团队）。

### 4.2 首页 · 课题组入口
**位置**：顶部「首页」。分三块：我的课题组、发现课题组、研究广场近期。
**能做**：任何成员可**创建课题组**（创建者自动成为课题组管理员）；对"发现课题组"里的组可**申请加入**（课题组管理员审批）。课题组卡片显示成员数、数据集数、我的角色。课题组管理员可关闭本组的"可发现性"，关闭后不在发现列表出现。

### 4.3 课题组详情
**位置**：点任一课题组卡片进入。含公约栏、数据集网格、成员、本组近期想法。
**权限**：进入时若未确认过本组公约会**弹窗要求确认**；组内成员可"发起新数据集"（必填发起人联系方式）；课题组管理员可编辑公约、审批加入。非成员只见公开信息（简介、成员数、被设为全平台可见的想法），不显示成员名单明细。

### 4.4 数据集详情（产品核心，7 个页签）
**位置**：点任一数据集卡片进入。顶部固定显示**发起人（可点击进主页）+ 联系方式**、数据集公约、是否敏感标记。发起人/管理员可点「编辑数据集」改元信息。

- **概览**：当前推荐版本、成员与加入审批记录（可点进每个成员主页）、近期刊物。管理员可在此移除成员。
- **数据看板**：① 样本描述性统计（从派生汇总表出图，条形图）② 自定义维度分布 ③ **AI / 手写描述分析**——在**只读沙箱**里执行分析代码，禁写回、禁网络，绝不修改原始值。**非成员也能看看板**。
- **版本库**：历史版本列表、changelog、当前版标记。发起人/管理员可**发布新版本**（上传 `.dta` + codebook），**版本不可覆盖**、旧版本永久保留。**下载分级**：成员下当前推荐版，管理员下全部历史版本，非成员只能下 codebook。
- **原始数据勘误**：成员提交勘误（可**上传证据附件**）→ 点开某条 bug 看详情，成员打 0–10 分 + 触发 AI 评分 → 管理员**终审**（采纳程度 + 最终分）→ 贡献按终审分加权。
- **处理代码库**：公开展示清洗/处理代码，可**上传代码文件或直接粘贴**；点开看源码，一键"生成数据处理说明"（AI）；支持代码勘误（同评分制审核）。
- **文献地图**：研究话题 + 重点文献清单（可带链接跳转），成员可添加文献、可"**AI 总结研究用途/话题**"。
- **数据核验**：规则 + AI 标出疑似问题（flag），成员**一键生成勘误草稿**进入评分制审核——**检测全自动，但绝不静默改原始数据**，人工只剩管理员发版时的一次批量确认。

### 4.5 研究广场（社区）
**位置**：顶部「研究广场」。发帖 + 想法流。
**能做**：发帖支持**可见性三级**（全平台 / 本课题组 / 仅自己）、加标签、关联数据集（带数据集标签的帖子会进入该数据集的"近期想法"）；点赞、评论（本人可删自己的评论）；本人可编辑/删除自己的帖子；总管理员/发起人可标"推荐讨论"。

### 4.6 我的主页（小红书式个人主页）
**位置**：顶部「我的主页」，或点任意用户名进入其主页。
**能做**：顶部蓝色 banner 只显示"贡献度"总分；四个页签——
- **在做的项目**：公开展示、欢迎讨论、可招募合作者。
- **我的贡献**：贡献明细（发起分、勘误按终审分加权、评审被采纳分、代码分等），每条可溯源。
- **个人简历**：模板或空白创建，支持大标题/小标题/正文/分点的**块级增删改**。
- **项目工作台**（私密协作）：仅选定成员可见，含进展记录、待办（可勾选完成/指派）、讨论存档、**结果文件上传**、Overleaf 链接跳转。

### 4.7 Skill 共享
**位置**：顶部「Skill 共享」。科研技能包（清洗流程、计量方法、AI 提示词、复现脚手架），**功能/权限与数据集同构**：可发起、有发起人与成员、可协作优化、可挂 GitHub 链接跳转；创建与优化**计入贡献度**。另有"GitHub 推荐 Skill"列表可点击跳转。

### 4.8 管理后台
**位置**：顶部「管理后台」，按你持有的角色展示不同内容。
- 总管理员：全部课题组（元信息）、总管理员管理与交接、**全站审计日志（仅动作元数据）**。
- 课题组管理员：加入课题组审批、公约编辑、**全员总贡献度 + 分数据集贡献度**（贡献视图归课题组管理员，不归总管理员，也不出现在首页/主页下方）。
- 数据集管理员：入口指向各数据集的"数据集后台"页签。

## 五、核心闭环：数据是怎么被越用越可信的

这是整个平台的价值主线，现已完整跑通：

**下载数据 → 使用中发现问题 → 提交勘误并附证据 → 成员评分 + AI 评分 → 管理员终审采纳 → 发布新版本时勾选修复的勘误（自动标记已修复并记录修复版本号）→ 旧版本永久保留、可复现 → 报告人贡献按终审分加权入账。**

三个设计要点让它区别于普通协作工具：一是**版本不可覆盖、更正必留痕**，任意时点可复现；二是**评分制审核**让"谁的勘误值多少分"有客观依据，直接换算成贡献；三是**代码与数据同等重要**——处理代码同样公开、可勘误、可归属。

## 六、AI 赋能：只发元数据、不碰原始数据

平台接入腾讯云 TokenHub（OpenAI 兼容），第一版启用的 AI 功能都遵循同一条安全红线——**只发送变量名、codebook、公开文本等元数据，绝不把原始敏感数据行发给模型**；涉及算数据的场景一律"AI 只写代码、代码在只读沙箱本地跑"。

| AI 功能 | 位置 | 作用 | 安全边界 |
|---|---|---|---|
| AI 勘误评分 | 勘误详情 | 给 bug 打 0–10 可采纳度分，辅助管理员终审 | 只发勘误描述与新旧值 |
| AI 生成分析代码 | 数据看板 | 自然语言 → pandas 分析代码 | 只发变量名/codebook；代码在只读沙箱跑，禁写回/禁网络 |
| AI 数据处理说明 | 代码详情 | 把处理代码转成论文"数据附录"写法草稿 | 只发代码文本 |
| AI 文献/用途总结 | 文献地图 | 总结数据集研究用途与话题 | 只发公开文献与数据集描述 |

模型可配置：默认快模型（评分/总结）+ 强模型（代码/写作），随时可在环境变量切换，业务代码零改动。出站还有脱敏网关拦截疑似原始数据表。**暂不做**的是让 AI 直接读原始数据自动改数——那始终走"发现→勘误管线"，人来把关。

## 八、数据安全与部署

平台从设计上支持"数据留在私有环境"：所有环境相关项走配置（数据库、存储、AI 网关、密钥、跨域），存储层是"本地目录 / 腾讯云 COS"双实现，切换只改配置。落地分两阶段——**第一阶段腾讯云先行**（快速上线、团队内测，只放合成/脱敏数据）；**第二阶段迁北大校内**承载真实敏感数据，只换配置、代码不变。无论哪个阶段：最小授权、下载分级、导出全程审计留痕、AI 网关脱敏一律生效。

技术上：后端 FastAPI + PostgreSQL + JWT，前端 Vue3 + Vite + TypeScript + Tailwind，Docker 一键部署，双语 i18n 全量。

---

## 九、一句话总结

平台是一个**多课题组的开放科研数据共享空间**：把各组自建数据变成可信、可复用、贡献可归属的共享资产（数据面），在其上长出跨组的低成本学术交流社区（社区面），再用三级权限、公约、评分制审核与清晰的贡献记录，把公共投入公平地兑现出来（治理面）。
