# 账号与安全规则

按 `api/auth.py`、`core/security.py`、`models/extras.py`（找回密码令牌）实际实现书写。

## 一、注册

- `POST /auth/register`，必填**用户名 + 密码 + 邮箱**。
- 唯一性：**用户名唯一**（"用户名已存在"）、**邮箱唯一**（"该邮箱已被注册"）。见 `10`。
- 显示名 `display_name` 缺省用用户名；**显示名允许重复**（唯一键是用户名/ID）。
- 新用户默认角色 `member`，注册即返回 access/refresh token。

## 二、登录与令牌

- 密码用 **argon2** 哈希（`PasswordHasher`）存储，绝不存明文。
- `POST /auth/login` 返回 JWT `access` + `refresh`（`create_access_token/refresh_token`，签名 `JWT_SECRET`）。
- `access` 携带 `sub`(用户id)/`type`/`iat`/`exp`；`decode_token` 校验 `type=="access"` 才放行受保护接口。
- `POST /auth/refresh` 用 refresh 换新 access。TTL 由 `JWT_ACCESS_TTL`/`JWT_REFRESH_TTL` 配置。
- 用户 `status=="left"` 视为已离开，令牌失效。

## 三、找回密码

- `POST /auth/forgot-password`：按邮箱找到用户 → 生成一次性 `PasswordResetToken`（`secrets.token_urlsafe`）→
  经邮件抽象层（见 `07`）发 `SITE_URL/#/reset-password?token=` 链接。**邮箱不存在时不报错**（防枚举）。
- `POST /auth/reset-password`：凭未使用的 token 重置密码，用后置 `used=True`。

## 四、鉴权下载的坑（工程约束）

需登录的下载接口（模板/导出/附件/skill/代码/版本文件/codebook/对照表等）**不能用 `window.open`**（不带 Bearer → 401）。
前端统一用 `utils/download.ts` 的 `downloadFile()`（axios blob + 拦截器带 token）。

## 五、资料与主页

- `PATCH /me`、`POST /me/avatar`、简历整块编辑 `PUT /me/resume/blocks`、在做项目 `POST /projects`（可置顶）。
- 全站按姓名/用户名/ID 检索用户：`GET /users/search?q=`。
