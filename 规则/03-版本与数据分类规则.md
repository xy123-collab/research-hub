# 版本与数据分类规则

按 `api/datasets.py`（发版/脱敏/应用勘误/变量抽取）、`models/curation.py`、`models/mapping.py`、
`services/data_ops.py`、`services/introspect.py` 实现书写。

## 一、版本不可覆盖

- `POST /datasets/{slug}/versions`：同一 `(dataset, version_id)` 重复报错「版本不可覆盖」。
- 旧版本永久保留；`PATCH .../versions/{vid}` 只能改 `changelog`，不能换文件。

## 二、三种数据分类（`data_kind`，`DATA_KINDS = raw|masked|sample`）

- **原始 raw** 与 **脱敏 masked**：参与迭代，竞争"当前推荐版" `is_current`。发新的 raw/masked 会把旧的
  `is_current` 置 False，并把新版设为 `d.current_version_id`。
- **样例 sample**：公开、独立、**不参与迭代**（不竞争 current）；所有登录用户可下载（见 `04`）。

## 三、每个版本可携带的文件

- 数据文件 `.dta`（`data_file_path`，仅 .dta）。
- **Codebook**（`codebook_file_path`，PDF/DOCX 等）。
- **对照表 / 取值字典**（新增，`VersionAuxFile.kind="mapping"`）：存数字编码↔中文/文字映射，如城市列表、职位列表；
  发版时 `mapping_file` + `mapping_note` 上传，可多个。`GET .../versions` 每条返回 `has_codebook` 与 `mappings[]`。

## 四、上传原始数据 → 自动抽取变量（不再写死变量）

- 发布 `data_kind=raw` 且带 .dta 时，自动 `sync_variables_from_version()`：
  `read_dta_schema()` 读列名 + Stata 变量标签，写入 `Variable` 表。
- 规则：已存在的变量**保留其中文标签与脱敏规则**（只在库里没填标签、而 .dta 带标签时回填）；
  新列新增；原库里已不存在的变量置 `enabled=False`（**不物理删除**，保住历史配置）。
- 手动补救：`POST /datasets/{slug}/variables/refresh` 从最新原始版重抽（仅管理员）。
- 变量清单驱动：数据处理设置的下拉、在线分析 df 的真实列（见 `09`）都来自这里。

## 五、一键脱敏（`services/data_ops.py`）

- `PUT /datasets/{slug}/data-config`：设唯一 ID 变量、`script_only`、每变量脱敏动作
  （`MASK_ACTIONS = keep|drop|hash|bucket`，bucket 带桶宽）。
- `POST .../versions/{vid}/desensitize`：从某原始版按规则生成脱敏版。
  **双路策略**：文件 ≤ 40MB 且非 script_only 时服务器直接生成脱敏 .dta 并发版；否则回退返回 Stata 脚本供本地运行。

## 六、批量勘误一键改数（见 `05`，与版本联动）

- `POST /datasets/{slug}/apply-corrections`：把"已采纳未应用"的勘误子项按 `唯一ID+变量名` 定位，
  自动改基准版 .dta → 生成新原始版本，子项标 `fixed`。**底层不写死变量名**，靠唯一 ID 变量 + 变量名规则定位。超限回退脚本。

## 七、内存与免费档注意

- pandas/openpyxl 在函数内**惰性导入**，不拉高常驻内存。数据处理走"≤40MB 服务器处理、否则脚本"双路。
- 沙箱/抽取读的是存储后端里的 .dta；**免费档无持久盘，重启会清空上传文件**，届时抽取/分析会提示"文件可能已丢失，需配置 COS"。
