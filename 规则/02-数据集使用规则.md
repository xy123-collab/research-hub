# 数据集使用规则

按 `backend/app/api/datasets.py` 与前端 `Dataset.vue` 实际实现，逐环节说明。

## 一、创建

- 两种入口：**独立创建** `POST /datasets`（`group_id=None`）；**组内创建** `POST /groups/{slug}/datasets`
  （须是该组成员/管理员，`group_role in (group_admin, member)`）。
- 校验：`slug` 全局唯一；**`name_zh` 全局唯一**（去空格+大小写不敏感，见 `10`）。
- 创建者即 `ds_role="founder"`（总管理员），自动建成员记录 + 数据集公约 `Charter` + 记贡献 30 分。
- 归属在创建时定死：已创建的数据集前端不再进出课题组（后端仍保留 `attach-request/detach-request` 接口备用）。

## 二、成员与加入

- 加入：`POST /datasets/{slug}/join-requests` → 管理员 `POST /join-requests/{rid}/decide?approve=`。
- **加入不自动给下载/在线分析等权限**（原则三）；这些走单独授权（见 `01`/`04`）。
- 成员展示：默认显示 3 个 + 检索弹窗（按姓名/ID）；都展示 ID 且链接主页。
- 移除成员 `DELETE /datasets/{slug}/members/{uid}`：仅管理员；不能移除总管理员；连带撤销其授权。

## 三、九个页签（进入数据集后）

| 页签 | 接口 | 作用 |
|---|---|---|
| 概览 overview | `GET /datasets/{slug}` | 当前版本、公约（可内联编辑）、发起人+联系方式、成员 |
| 更新记录 activity | `GET /datasets/{slug}/activity?kind=` | 版本/勘误/代码时间线，可筛选 all/version/bug/code |
| 数据看板 dashboard | `GET .../dashboard`、`.../analysis/*` | 描述性统计 + AI/手写只读沙箱分析（见 `09`） |
| 版本库 versions | `GET/POST .../versions` | 发版、下载、数据分类、脱敏、应用勘误、数据处理设置、codebook/对照表（见 `03`/`04`） |
| 勘误 bugs | `api/bugs.py` | 单条/批量勘误、逐条终审、应用改数（见 `05`） |
| 处理代码 code | `api/code.py` | 代码版本迭代、授权、下载、评论/勘误（见 `05`） |
| 文献地图 literature | `.../literature*` | AI 综述 + 词云 + 文献清单，上传三选一 + AI 核验 + 同集查重（见 `10`） |
| 研究讨论区 feed | `GET /posts?dataset_id=` | 该数据集相关帖子 |
| 成员与权限 access（仅管理员） | `GET .../members` | 审批、授权、增删管理员、成员管理 |

## 四、数据集设置与关闭

- `PATCH /datasets/{slug}/settings`：`download_policy`、`history_visible`、`history_downloadable`、
  `analysis_open`、`codebook_public`、`dashboard_public`（`DatasetSettings`，懒创建）。见 `04`。
- `POST /datasets/{slug}/close`：关闭后不再接受加入/勘误/发版（`is_closed`）。
- `PATCH /datasets/{slug}`：编辑名称/简介/图标/联系方式/敏感标记；改名同样查重。

## 五、编辑约束

- 仅管理员可发版/编辑/设置。版本文件**不可覆盖**，只能改 changelog 元数据。
- 唯一 ID 变量在「数据处理设置」指定，用于批量勘误定位（其本身不可被勘误修改）。
