# 可见范围规则

按 `backend/app/core/scopes.py` 与 `models/extras.py` 的 `ContentScope` 实际实现书写。
统一供**发帖、创建项目、发起 Skill** 三处使用（前端 `ScopeSelector.vue`）。

## 一、四类可见范围（`CONTENT_SCOPES`）

| scope | 中文 | 判定 |
|---|---|---|
| `public` | 全平台公开 | 所有登录用户可见 |
| `group` | 课题组成员可见 | 支持**多选**：是所选任一课题组成员即可见 |
| `dataset` | 数据集成员可见 | 支持**多选**：是所选任一数据集成员即可见 |
| `self` | 仅自己可见 | 仅作者本人（+ 平台管理员，见下）可见 |

## 二、写入与校验（发布时）

- `validate_scope()`：group/dataset 必须选至少一个 ref；且发布者**必须是所选组/集的成员**，否则报错。
- `set_scope()`：public/self 存一行（ref=None）；group/dataset **每个 ref 存一行**（多选=多行）。

## 三、可见性判定（`scope_visible()`）

顺序：
1. 作者本人 **或 平台管理员** → 可见（`owner_id==user.id or is_super_admin`）。
   ——注意：这是可见范围里**唯一**给 super_admin 的例外（仅"看内容列表"，不等于数据集内容权限，见 `01` 原则一）。
2. 无 scope 记录 → 默认公开（兼容历史数据）。
3. 含 `public` → 可见；仅 `self` → 不可见。
4. group/dataset → 命中任一 ref 的成员身份即可见。

列表接口据此过滤；`scope_summary()`/`scope_label()` 给前端展示（多选显示"（N个）"）。
`GET /me/collab-scopes` 提供"我所在的组/集"下拉。
