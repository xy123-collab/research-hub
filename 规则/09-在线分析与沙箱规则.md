# 在线分析与沙箱规则

按 `services/sandbox.py`、`services/introspect.py` 与 `api/datasets.py`（analysis 三个接口 + ai-hint）实际实现书写。

## 一、谁能用（授权，`_require_analysis`）

需**先加入数据集**；然后满足其一：是数据集管理员 / 数据集设了 `analysis_open=True` / 获 `analysis.online` 单独授权。
否则 403。前端未授权处提供"申请开通在线分析"。

## 二、沙箱里的 df = 真实原始数据（本轮改动，接真实变量）

- `GET /datasets/{slug}/analysis/context`：返回是否连上原始数据 + 真实变量清单（`_sandbox_variables`：优先 `Variable` 表，回退读最新原始版 .dta 列）。
- `POST .../analysis/run`：加载**最新「原始」版**真实 .dta（`load_dta_records`）为 `df`，另注入 `columns`。
  - **行数上限 5 万**（`SANDBOX_MAX_ROWS`）防爆内存；超出只读前 5 万行并在结果里标 `truncated`。
  - 没有原始版 → 400"请先发布原始数据"；文件读不到 → 400"可能已丢失，需配置 COS"。
- 列名就是真实变量名，`df['变量名']` 直接可用（这是本轮修复的核心：之前是派生汇总长表、且被清空）。

## 三、只读隔离与禁止操作（`sandbox.py`）

- 在独立 `multiprocessing` 子进程执行，**10 秒超时**。
- **静态过滤** `_FORBIDDEN`：`open( / to_csv / to_stata / to_pickle / write / os. / sys. / subprocess /
  socket / requests / urllib / shutil / remove / __import__ / eval( / exec(` → 直接拒绝（禁写回、禁网络）。
- 受限 builtins（只给 len/range/min/max/sum/round/sorted/abs/list/dict/set/str/int/float/bool/enumerate/zip/tuple/print）。
- **绝不写回原始数据**——这是"只读沙箱"的技术保证。
- `strip_code_fences`：先剥掉 AI 返回的 ```python ... ``` markdown 围栏再执行（修复"invalid syntax"）。
- 结果 `_jsonable` 转成可 JSON 的普通对象。

## 四、AI 只送元数据，绝不上传原始数据行

- `POST .../analysis/generate`：只把**变量名/需求**发给 AI，返回 pandas 代码；不发数据行。AI 未启用/未连数据时返回友好 `note`，不报错。
- **AI 勘误助手** `POST /datasets/{slug}/ai-hint`（左下角悬浮精灵）：`check` 结合预设方向+变量清单+人工提示，
  **只提示"哪些方面可能有问题、请人工核查"，绝不读原始数据、不自动改数**；`patterns` 从已有勘误归纳高频错误。点一次跑一次。
- 文献 AI 综述/核验/引文匹配同理：**只送标题/元数据**。
- 所有 AI 依赖 Render 环境变量 `AI_API_KEY`（TokenHub）；未配置时返回"AI 未启用"占位并**不阻断**流程。
