# 名称与文献去重规则

按 `backend/app/core/naming.py`、`api/datasets.py`、`api/groups.py`、`api/auth.py` 实际实现书写。
比较口径统一：`normalize_name` = 去首尾空格 + 连续空白压成一个 + `casefold` 小写化；库里存原样输入，只在判重时归一化。

## 一、名称必须唯一（创建/重命名时自动检索，重复即拦截「已存在」）

| 名称 | 范围 | 落地位置 |
|---|---|---|
| 账号用户名 `username` | 全局 | `api/auth.py` 注册 |
| 注册邮箱 `email` | 全局 | `api/auth.py` 注册 |
| 数据集机器名 `slug` | 全局 | 创建数据集 |
| **数据集名称 `name_zh`** | 全局（未软删） | 独立建集 / 组内建集 / `PATCH /datasets/{slug}` 改名 |
| 课题组机器名 `slug` | 全局 | 创建课题组 |
| **课题组名称 `name_zh`** | 全局（未软删） | 创建 / 编辑课题组 |
| 版本号 `version_id` | 同一数据集内 | 发版（"不可覆盖"） |
| 变量名 `var_name` | 同一数据集内 | 由 .dta 抽取，天然唯一 |

重命名时按主键 id 排除自身（`ensure_unique(..., exclude_id=)`），改回原名或仅大小写微调不会误报。

## 二、名称允许重复（不拦截）

- **用户显示名 `display_name`**：同名同姓正常，唯一键是用户名 / ID。
- 帖子标题、评论、工作台条目、公约、changelog、对照表说明等自由文本。
- 处理代码 / Skill 的标题：版本化资源，靠 ID 区分，允许重名。

## 三、文献去重（同一数据集内，四项全一致才算重复）

- 判重签名 `_lit_sig = (标题, 作者, 年份, 刊物)`，四项都归一化。**只有四项全一致**才算重复
  （仅标题相同、作者/年份/刊物任一不同 → 不算重复，允许上传）。
- 落地：
  - 单条 `POST /datasets/{slug}/literature/refs`：命中重复返回 `{ok:false, duplicate:true}`，前端提示"已存在完全一致的文献"。
  - 批量 `POST /datasets/{slug}/literature/batch`：库内已有的跳过，**本批次内**四项相同的也只留一条，返回 `skipped_duplicates`。
- **文献显示名/标题本身允许重复**——不同数据集常引同一篇；同集内也只有四项全同才拦。

## 四、前端提示

创建/改名/文献接口返回明确中文（如"数据集名称「X」已存在，请换一个"、"本数据集内已存在标题、作者、年份、刊物完全一致的文献"）。
`Home.vue`、`Groups.vue`、文献上传浮层都会把该提示 `alert` 给用户。
