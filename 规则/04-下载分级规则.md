# 下载分级规则

按 `api/datasets.py` 的 `check_download()` / `download()` 与 `DatasetSettings` 实际实现书写。
下载入口：`GET /datasets/{slug}/versions/{vid}/download?file=data|codebook|mapping`。

## 一、判定顺序（数据文件 file=data，`check_download`）

1. **样例数据（sample）**：直接放行——所有登录用户可下（`return True, "sample"`）。
2. **数据集管理员**：直接放行。
3. **非成员**：拒绝（"非成员不能下载原始数据，请先申请加入"）。
4. **历史版本**（`!is_current`）：需 `download.history` 授权，或数据集设了 `history_downloadable`；否则拒绝。
5. **当前推荐版**：按数据集 `download_policy` 五选一（下节）。

## 二、五种下载策略（`DOWNLOAD_POLICIES`，`DatasetSettings.download_policy`）

| 策略 | 含义 |
|---|---|
| `public` | 成员直接可下（当前版） |
| `member`（默认） | 成员直接可下 |
| `approval` | 需 `download.current` 授权，或"下载申请已批准且覆盖该版本"，否则拒绝 |
| `masked_only` | 只提供脱敏数据下载，需 `download.masked` 授权 |
| `forbidden` | 关闭下载通道，仅可用在线分析 |

## 三、下载申请审批（approval 策略下）

- `POST /datasets/{slug}/download-requests`：填研究用途、申请版本、预计使用时间、是否与他人共享、是否同意公约。
- 管理员 `POST /download-requests/{rid}/decide?approve=`（可设有效期 `valid_to`）；`/revoke` 撤销。
- `_has_approved_download()`：存在 status=approved、未过期、覆盖该版本的申请即放行。

## 四、Codebook 与对照表下载（file=codebook / mapping）

- 与数据分级**分开**：成员可下；非成员仅当 `codebook_public=True` 时可下（对照表沿用同一开关）。
- 对照表取该版本最新一条 `VersionAuxFile(kind="mapping")`。

## 五、单独授权的到期范围（`DatasetGrant`，见 `01`）

`permanent` / `until_date`（到 `valid_to`）/ `version`（仅指定版本）/ `project`（指定项目）。同权限先撤旧再发新。

## 六、留痕

每次下载写 `DownloadLog`（user/dataset/version/file_type/file_name/时间）+ `AuditLog`（含权限来源，如 `policy:member`、
`grant:download.current`、`download_request`、`sample`、`admin`）。对应原则六。
