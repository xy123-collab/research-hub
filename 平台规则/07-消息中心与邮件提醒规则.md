# 消息中心与邮件提醒规则

按 `api/notifications.py`、`api/notify_prefs.py`、`services/digest.py`、`services/notify.py`、
`services/weekly_digest.py`、`components/NotificationCenter.vue`、`views/Profile.vue`（通知设置）实际实现书写。

**核心原则（对齐《邮件通知与每周周报功能需求文档》）**：程序决定谁能收到、哪些内容可发送；
AI 只做文本摘要，不决定收件人与权限。所有邮件都遵守课题组/数据集/帖子的可见范围；
平台总管理员不因身份自动收到内部内容。

## 一、消息中心（右下角悬浮铃铛）

- 入口：`App.vue` 挂载的 `NotificationCenter.vue`，右下角悬浮 + 红色数字角标（未读数，>99 显示 `99+`）。
- 拉取：`GET /notifications`，前端每 **60 秒**轮询一次。按当前用户**实时角色聚合**，分五类：

| category | 名称 | 计红点 | 例子 |
|---|---|---|---|
| `todo` | 待我处理 | 是 | 加入审批、下载审批、勘误终审、codebook/对照表勘误确认、权限申请、归属审批 |
| `access` | 权限与身份 | 否 | 我的加入/下载被处理、被设/撤管理员 |
| `interact` | 互动·评论 | 否 | 帖子评论、代码评论、**被@（提及）** |
| `collab` | 协作·工作台 | 否 | 被加入工作台 |
| `publish` | 发布·数据与代码 | 否 | 我参与数据集出新版本、新代码 |

- 未读游标 `notification_state.last_read_at`；「全部已读」(`POST /notifications/mark-read`) 推进游标。
- **被@（type=mention）**：`services/mentions.mentions_for_user` 计算——直接@我、或@了我所在的
  数据集/课题组（整组），都进入 `interact` 类，属于新消息提示范围（见 `13`）。

## 二、通知偏好（细分订阅）

- 存 `notification_preferences`（每用户一行，懒创建，缺省即默认全开）。接口 `GET/PUT /notification-preferences`。
- 入口：个人主页「编辑资料」（总开关）与独立「**通知设置**」弹窗（细分）。每封邮件底部附退订/设置链接。
- **总开关 `email_enabled`**：与找回密码**隔离**——找回密码邮件始终发送，不受此开关约束。
- 开启后可细分：

| 字段 | 含义 | 频率选项 |
|---|---|---|
| `version_email_enabled` / `_frequency` | 所在数据集**数据版本更新** | `immediate` 立即 / `daily` 每日8:00 / `weekly` 每周一8:00 |
| `code_email_enabled` / `_frequency` | 所在数据集**处理代码更新** | 同上 |
| `message_email_enabled` | **消息通知**（被回复/被@/权限申请/权限通过等） | 每日 8:00 与 18:00 巡检，有新则发，已发过不再重发 |
| `weekly_digest_enabled` / `_scope` | **每周帖子周报** | 范围多选：`public` 全平台公开 / `groups` 我的课题组 / `datasets` 我加入的数据集 |
| `email_language` | 邮件语言 | `zh-CN` / `en` |

## 三、功能 A：数据版本更新邮件（`services/notify.notify_version_published`）

- **触发**：数据集管理员**正式发布新版本**事务提交成功后（`api/datasets.publish_version` 末尾）。
  上传候选、存草稿、改简介、预览等**不触发**。通知创建失败不影响发版（try/except 回滚）。
- **收件人**（`recipients_for_version`）：数据集管理员 + 成员 + 有效下载授权用户 + 关注且开启版本通知者；
  **排除**：已退出、账号 left、无邮箱、关闭了版本通知、仅总管理员身份但未加入者。
- **权限展示**：对每个收件人用 `check_download` 判定，邮件显示「你可以下载该版本」或「仅可查看版本说明」。
- **内容**：数据集名/新版本号/发布时间/发布者/是否推荐版/更新摘要/权限说明/进入链接/退订入口。
- **频率**：`immediate` 立即发；`daily`/`weekly` 落 pending，由到点任务 `flush_due_deliveries` 汇总合并发送。

## 四、功能 A'：处理代码更新邮件（`notify_code_published`）

- 触发：`api/code.publish_code_version` 成功后。收件人为该数据集成员中开启 `code_email_enabled` 者。频率同上。

## 五、功能 B：每周帖子周报（`services/weekly_digest.run_weekly_digest_once`）

- 默认**每周一 8:00** 汇总上一自然周（周一~周日）。时间由 `WEEKLY_DIGEST_DOW/HOUR` 环境变量配置。
- **权限过滤在调用 AI 之前完成**：先按每个收件人的 `scope_visible` 与其 `weekly_digest_scope` 选择，
  筛出可见新帖，再把标题/作者/互动数/摘要等**元数据**交 AI 生成摘要；AI 不可用/失败时降级规则式（按互动排序列标题）。
- 不纳入：仅自己可见、无权查看、已删除、草稿、超周期。AI 不决定收件人、不绕过可见性、不上传敏感附件。

## 六、去重 / 留痕 / 重试（`email_deliveries` + `notification_events`）

- **事件** `notification_events`：一次“发生的事”（版本发布/代码发布/周报/周期）。
- **投递** `email_deliveries`：每收件人一条，`dedupe_key` 唯一 → 同用户+同对象+同类型只成功发一次；
  记录 status（pending/sending/sent/failed/skipped）、scheduled_at、sent_at、retry_count、error_message。
- **重试**：`retry_delivery`；管理后台 `POST /admin/notifications/deliveries/{id}/retry`（仅总管理员）。
- **管理后台**：`GET /admin/notifications` 看事件与发送统计，`GET /admin/notifications/{eid}/deliveries` 看逐条状态。
  总管理员可看发送状态，但邮件正文只放摘要，不借此读内部内容。

## 七、退订（签名令牌，`services/notify.make_unsub_token` / `GET /unsubscribe`）

- 邮件底部退订链接带 **HMAC 签名 + 30 天过期** 令牌，不暴露 user_id/邮箱明文。
- 点击后置 `email_enabled=False`（同步旧 `UserProfile.email_opt_in`）；找回密码等必要邮件不受影响。

## 八、定时任务（`main.py` APScheduler，本地时区 `DIGEST_TZ`）

- 每日 `DIGEST_HOURS`（默认 8/18）:00 跑消息摘要 `run_digest_once`（尊重 `message_email_enabled`）；
  :05 跑 `flush_due_deliveries` 发出到点的版本/代码汇总。
- 每周一 8:00 跑 `run_weekly_digest_once`。
- 手动触发：`POST /api/admin/run-digest`、`/run-weekly-digest`、`/flush-deliveries`。

## 九、邮件抽象层（`core/email_service.py`）

- `EMAIL_BACKEND`：`mock`（默认，只落 `email_events`+日志，不真发）/ `smtp`（真发）/ `none`。不强绑第三方，可接校内 SMTP。
- ⚠️ Render 免费档封了出站 SMTP 端口，`smtp` 会报 "Network is unreachable"；真发需升级付费实例或部署校内服务器
  或换 HTTP 邮件 API。API key 只放后端环境变量，绝不进前端/Git。
