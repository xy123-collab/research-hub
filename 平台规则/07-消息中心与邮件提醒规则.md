# 消息中心与邮件提醒规则

按 `api/notifications.py`、`services/digest.py`、`components/NotificationCenter.vue` 实际实现书写。

## 一、消息中心（右下角悬浮铃铛）

- 入口：`App.vue` 挂载的 `NotificationCenter.vue`，右下角悬浮 + 红点角标。
- 拉取：`GET /notifications`，前端每 **60 秒**轮询一次（`setInterval(load, 60000)`）。
- 内容按当前用户**实时角色聚合**，分五类（`CATEGORY_META`）：

| category | 名称 | 计红点 | 例子 |
|---|---|---|---|
| `todo` | 待我处理 | 是 | 加入审批、下载审批、原始数据勘误终审、**codebook/对照表勘误确认**、权限申请、归属审批 |
| `access` | 权限与身份 | 否 | 我的加入/下载被处理、被设/撤管理员 |
| `interact` | 互动·评论 | 否 | 帖子评论、代码评论 |
| `collab` | 协作·工作台 | 否 | 被加入工作台 |
| `publish` | 发布·数据与代码 | 否 | 我的数据集出新版本、新代码 |

- 点条目直达对应页面（数据集/课题组/帖子等）。`todo` 只对**有相应管理权限**的人显示（管理员看到审批类）。

## 二、每日摘要邮件（`services/digest.py`）

- 默认每天 **8:00 与 18:00**（`DIGEST_HOURS="8,18"`，时区 `DIGEST_TZ="Asia/Shanghai"`）APScheduler 巡检。
- 给"有新消息"的用户发摘要；**内容哈希去重**（较上次有变化才发），避免重复打扰。
- 开关 `DIGEST_ENABLED`；手动触发 `POST /api/admin/run-digest`。

## 三、邮件抽象层（`core/email_service.py`）

- `EMAIL_BACKEND`：`mock`（默认，只落 `email_events` + 日志，不真发）/ `smtp`（填 `SMTP_*` 真发）/ `none`。
- **不强绑第三方**，可接校内 SMTP。相关 config：`EMAIL_FROM`、`SMTP_HOST/PORT/USER/PASSWORD/TLS`、`SITE_URL`。

## 四、未读计数与互动通知（本轮新增，详见 `11-研究讨论区规则`）

- 悬浮消息入口显示**红色圆形数字角标**（未读数，>99 显示 `99+`），不再只是红点。
  角标 = 待我处理（待办）+ 互动未读数（`GET /notifications` 的 `badge_count`）。
- 互动通知新增：评论我的帖子、回复我的评论、点赞我的帖子（按帖子合并）、关注我的帖子。
- 未读游标 `notification_state.last_read_at`：晚于游标的互动消息计未读；打开面板不自动清零，
  「全部已读」(`POST /notifications/mark-read`) 把游标推进到当前时间才清零；跨设备/刷新一致。
- 待办类（审批）在处理前始终保留；点赞短时合并，评论/回复/审批不合并；删帖后相关通知自然失效。
