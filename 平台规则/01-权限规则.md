# 权限规则（完整）

严格按 `backend/app/core/permissions.py` 与 `backend/app/models/access.py` 实现书写。
所有权限一律按**具体某课题组 / 某数据集**判定，一个用户可同时有多种身份。

## 一、思维导图

```
权限体系
├── 平台级（只管系统，不碰内容）
│   └── 平台管理员 super_admin（User.role.code == "super_admin"）
│       ├── 平台总管理员 primary   （platform_settings.primary_super_admin_uid）
│       │   ├── 可交接头衔 transfer（只有 primary 能做，交出后自己降为"其他管理员"）
│       │   └── 可移除其他管理员（不能移除自己；要退位须先交接）
│       └── 其他平台管理员          （任一平台管理员都能"新增"一名）
│       └── ★原则一：平台管理员≠数据管理员，绝不自动获得任何组/集的内容权限
│
├── 课题组级（GroupMember.group_role）
│   ├── 课题组总管理员 group_owner  （GROUP_LEAD_ROLES）——组内唯一
│   │   └── 独占：增/删管理员、转让总管理员、编辑组、审批加入、移除成员
│   ├── 课题组管理员 group_admin    （GROUP_ADMIN_ROLES = owner+admin）
│   └── 课题组成员 member
│
├── 数据集级（DatasetMember.ds_role）
│   ├── 数据集总管理员 founder/owner （DS_LEAD_ROLES）——全集唯一，founder=历史命名=owner
│   │   └── 独占：增/删管理员、转让总管理员
│   ├── 数据集管理员 admin           （DS_ADMIN_ROLES = founder+owner+admin）
│   │   └── 天然拥有全部数据集权限；可授权/审批/发版/删版本/设置/勘误终审…
│   └── 数据集成员 member
│       └── ★原则三：加入≠下载权，默认无下载/在线分析等，需单独授权
│
└── 普通用户（已登录、不属于任何组/集）
```

## 二、四条核心原则（代码层落地）

- **原则一 平台管理员 ≠ 数据管理员**：`is_super_admin()` 只用于平台系统路由（`require_super_admin`）；
  `is_dataset_member/admin`、`is_group_member/admin` 等内容判定**都不看 super_admin**。总管理员想看内容须走正常加入流程。
- **原则二 加入课题组 ≠ 加入数据集**：`GroupMember` 与 `DatasetMember` 两套表分别判定。
- **原则三 加入数据集 ≠ 下载权**：成员默认无下载/在线分析等权限，需 `DatasetGrant` 单独授权。
- **原则四~六**：留痕（`AuditLog`/`DownloadLog`）、审批、公约由各接口配合 helper 执行。

## 三、两级管理员——增/删/转让/个数约束

### 数据集（`api/datasets.py`）
- 总管理员 = `ds_role in (founder, owner)`，**全集唯一**；`dataset_lead_id()` 优先取该角色，回退 `founder_id`（兼容历史）。
- **仅总管理员**能：`POST /datasets/{slug}/admins/{uid}`（设为 admin）、`DELETE /datasets/{slug}/admins/{uid}`（降级）、
  `POST /datasets/{slug}/transfer-lead/{uid}`（转让总管理员）。
- **总管理员不能被移除成员**（`不能移除总管理员；请先转让`）；总管理员要退位必须先 `transfer-lead`（不能直接自撤）。
- 移除成员会**连带撤销其全部单独授权**（`DatasetGrant.revoked=True`）。

### 课题组（`api/groups.py`）
- 总管理员 = `group_role == group_owner`，组内唯一；`group_lead_id()` 优先取 owner，回退 `created_by`。
- **仅总管理员**能增/删管理员、转让（`/groups/{slug}/admins/{uid}`、`/groups/{slug}/transfer-lead/{uid}`）。
- `count_group_admins()` / `count_dataset_admins()` 用于"至少保留一名管理员"类保护。

### 平台管理员（`api/admin.py`）
- `platform_settings.primary_super_admin_uid` 记录 primary。
- 任一平台管理员可 `POST /admin/super-admins`（新增）；
- **仅 primary** 可 `POST /admin/super-admins/transfer`（交接头衔，交出后自己变"其他管理员"）与
  `DELETE /admin/super-admins/{uid}`（移除其他管理员，**不能移除现任 primary/自己**）。
- 未指定 primary 时默认取最早的一位，避免无人可交接。

## 四、八项「可单独授予」的权限（`GRANTABLE_PERMS`）

| 权限码 | 中文 | 体现在哪个功能 |
|---|---|---|
| `download.current` | 下载当前完整正式版本 | 版本库下载 .dta（当前版），审批策略下也认此授权 |
| `download.masked` | 下载脱敏数据 | `masked_only` 策略下下载 |
| `version.history.view` | 查看历史版本 | 历史版本可见 |
| `download.history` | 下载历史版本 | 非当前版本下载 |
| `analysis.online` | 在线分析功能 | 数据看板 AI/手写沙箱运行（见 `09`） |
| `upload.candidate` | 上传版本候选文件 | `POST /datasets/{slug}/candidates` |
| `codebook.draft` | 编辑 codebook 草稿 | `PUT /datasets/{slug}/codebook-draft` |
| `code.maintain` | 上传和维护处理代码 | 处理代码库上传/维护 |

管理员天然拥有全部（`has_dataset_perm` 里 `is_dataset_admin` 直接 True）。

## 五、授权的到期规则（`DatasetGrant.scope_type`）

- `permanent` 永久；`until_date` 到 `valid_to` 失效；`version` 仅对 `scope_version` 指定版本有效；
  `project` 仅用于指定研究项目（`project_note`）。
- 同一权限**先撤旧再发新**，不叠加。`is_active(now)` 判到期。

## 六、成员申请与审批入口（按钮在哪）

- 数据集「成员与权限」页签（管理员）：审批加入 `POST /join-requests/{rid}/decide`、审批下载
  `POST /download-requests/{rid}/decide`、单独授权 `.../members/{uid}/grant` 与 `/revoke`、
  增删管理员、移除成员、审批权限申请 `POST /perm-requests/{rid}/decide`。
- 成员侧：无某权限处有「申请开通…」（`POST /datasets/{slug}/perm-requests`）；
  需下载时提交下载申请（`POST /datasets/{slug}/download-requests`，含研究用途/版本/预计时间/是否共享/是否同意公约）。
- 课题组：`/groups/{slug}/join-requests`、`/group-join/{rid}/decide`、成员管理、编辑组。
- 平台后台：`Admin.vue` 用 `/users/search`（按姓名或 ID）检索后新增/交接/移除平台管理员。

所有授权、审批、发版、下载、删除等敏感动作都写 `AuditLog`（留痕，原则六）。
