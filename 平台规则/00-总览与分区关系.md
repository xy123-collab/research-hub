# 分区关系规则（平台骨架）

严格按 `frontend/src/router/index.ts` 与 `frontend/src/App.vue` 的实际路由/导航书写。

## 一、顶部导航 = 五大分区 + 个人入口

`App.vue` 顶栏固定五个主入口，右上角是个人主页：

- **数据集（首页）** `/` → `Home.vue`：进来直接是数据集网格（我参与的 + 发现）。这是平台的中心。
- **课题组** `/groups` → `Groups.vue`：课题组列表；详情 `/groups/:slug` → `Group.vue`。
- **研究讨论区** `/feed` → `Feed.vue`：全平台/范围内的讨论帖。
- **其他协作** `/collab` → `Collab.vue`：Skill 协作 + 用户自建协作分区。（旧 `/skills` 重定向到这里。）
- **管理后台** `/admin` → `Admin.vue`：管理控制台（按我管理的范围切换）。
- **我的主页**（右上角头像+用户名+ID）`/me` / `/users/:id` → `Profile.vue`。

登录/找回密码：`/login`、`/reset-password`（不需登录）。其余路由都需登录态。

## 二、五大分区之间的关系

```
                         ┌─────────────────────────┐
                         │        用户 / 账号        │
                         │ 登录后拥有下列身份的任意组合 │
                         └───────────┬─────────────┘
                                     │
        ┌───────────────┬───────────┼───────────────┬────────────────┐
        ▼               ▼           ▼               ▼                ▼
   ┌─────────┐    ┌──────────┐  ┌────────┐    ┌──────────┐    ┌──────────┐
   │  数据集   │    │  课题组   │  │研究讨论区│    │ 其他协作  │    │ 管理后台  │
   │ (核心)   │    │(可选归属) │  │ (发帖)  │    │(Skill等) │    │(控制台)  │
   └────┬────┘    └────┬─────┘  └────┬───┘    └────┬─────┘    └────┬─────┘
        │              │             │             │               │
  版本/勘误/代码/    组内多个数据集    帖子可关联     可见范围四选一    只按"我管理的
  文献/看板/成员权限  组内动态/成员    某数据集超链接  (见06)          组/集"操作(见01)
        │              │
        └──── 数据集可"独立"存在，也可在创建时归属一个我已加入的课题组 ────┘
             （已创建的数据集不再在前端进出课题组；归属在创建时定死）
```

关键关系（均由代码约束）：

1. **数据集是中心，课题组是可选容器**。`datasets.group_id` 允许为空（独立数据集）。
   创建数据集时可选填归属课题组，且只能选**我已加入**的组（`api/groups.py` 的组内建集校验 `group_role`）。
2. **加入课题组 ≠ 加入数据集**（原则二）。两套成员表分别判定，见 `01-权限规则`。
3. **研究讨论区 = 统一帖子系统**：研究广场、数据集讨论区、课题组讨论、个人主页"我的讨论"
   共享同一份帖子数据与交互规则，只是默认关联与筛选不同（详见 `11-研究讨论区规则`）。
   帖子可关联某数据集/课题组（超链接），可见范围走统一四类（见 `06`）。
4. **其他协作**里的 Skill/自建分区，可见范围同样走统一四类。
5. **管理后台**不是"平台上帝视角"：它按 `GET /admin/my-scopes` 返回**我管理的**课题组/数据集来切换，
   平台系统标签（审计、总管理员交接）仅平台总管理员可见（见 `01`）。

## 三、个人主页 / 工作台（挂在个人入口下，非顶栏分区）

- 个人主页 `Profile.vue`：编辑资料（头像/姓名/研究方向/关键词）、整块文本简历、在做项目（可置顶）。
- 工作台 `api/workspaces.py`：时间轴条目（进展/数据/图表/文献/结果/讨论/其他）+ 成员邀请，属于个人/小组协作空间，不在五大分区里单列。

## 四、数据集详情页的页签（进入某数据集后）

`Dataset.vue` 的 `tabs`：**概览 · 更新记录 · 数据看板 · 版本库 · 勘误 · 处理代码 · 文献地图 · 研究讨论区**；
额外的 **成员与权限** 页签仅数据集管理员可见（`d.is_admin`）。各页签逻辑见 `02-数据集使用规则`。
