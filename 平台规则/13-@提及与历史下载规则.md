# @提及与历史下载规则

按 `services/mentions.py`、`api/mentions.py`、`services/downloads.py`、`api/downloads.py`、
`components/MentionInput.vue`、`components/DownloadHistoryModal.vue` 实际实现书写。

## 一、@提及（Mention）

### 可@范围
平台所有可评论处（帖子评论、项目评论、处理代码评论、Skill 评论）都支持 @。可@对象仅限：

1. 自己所在的**全部数据集/课题组**（可@整组，即通知该数据集/课题组全体成员）；
2. 上述数据集/课题组**成员范围内的单个成员**。

### 检索
`GET /mentions/candidates?q=&scope_type=&scope_id=`：返回我所在的数据集/课题组（可@整组）
与范围内成员，`q` 关键词匹配名称/用户名/ID，可用 `scope_type+scope_id` 限定某一个具体数据集/课题组。
前端 `MentionInput.vue`：输入 `@` 触发下拉，选中后把 `@名称` 写入文本并记录结构化 mention。

### 写入与校验
- 评论创建时前端把结构化 `mentions=[{target_type,target_id}]` 一并提交
  （帖子/项目/Skill 走 JSON，代码评论走 Form 的 `mentions` JSON 串）。
- 后端 `record_mentions` **按发帖人权限二次校验**：只有 user 属于我的数据集/课题组成员、
  dataset/group 属于我所在范围时才落 `mentions` 表，越权的被丢弃。target_type ∈ {user, dataset, group}。

### 通知
- `mentions_for_user`：被@我本人、或@了我所在数据集/课题组（整组）的记录，进入消息中心 `interact` 类，
  属于新消息提示范围；`mentioned_by==自己` 的不通知自己。也随每日消息摘要邮件发出（见 `07`）。

## 二、历史下载（DownloadHistory）

- 统一表 `download_history`：`source`（dataset_version/code/bug_attachment/skill/post_attachment）、
  `dataset_id`（可空）、`location_label`、`detail`、`file_name`、`link`、`downloaded_at`。
- 所有下载端点调用 `services/downloads.log_download` 写入（失败不影响下载本身）。
- `GET /me/downloads`：本人下载记录，可按 `dataset_id`/`source` 筛选，返回下载时间 + 文件名 + 位置 + 说明 + 跳转。
- 双入口指向同一视图：**数据集页**快捷条（默认只看本数据集，可切全部）与**个人主页**「历史下载」。
