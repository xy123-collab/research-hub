# 11 · 研究讨论区规则（统一帖子系统 / 热榜 / 计数 / 未读）

> 本文对应本轮"讨论区统一重构"。核心思想：**研究广场是唯一一套讨论系统**，
> 数据集讨论区、课题组讨论、个人主页"我的讨论"只是从不同上下文进入同一套帖子，
> 自动套用不同的默认关联与筛选，而不是各自维护一套功能不一致的模块。

## 一、统一的帖子系统（一份数据，多处入口）

所有帖子都存在 `posts` 表，评论存 `post_comments`，点赞存 `post_reactions`，
标签存 `post_tags`，附件存 `post_attachments`，关注存 `post_follows`，可见范围走统一
`content_scopes`（见 `06-可见范围规则`）。前端四个入口共用同一批接口与两个组件
（`components/PostCard.vue` 帖子卡片、`components/PostComposer.vue` 发布/编辑弹窗）：

| 入口 | 页面 | 默认关联 | 备注 |
|---|---|---|---|
| 研究广场 | `Feed.vue` | 用户自选 | 三栏信息流，全站可见范围内的帖子 |
| 数据集讨论区 | `Dataset.vue` feed 页签 | 当前数据集 | 仍可补充关联其他集/组 |
| 课题组讨论 | `Group.vue` 组内讨论 | 当前课题组 | |
| 个人主页·我的讨论 | `Profile.vue` discuss 页签 | 不预设 | 展示作者本人的帖子 |

在任意入口对同一帖子的编辑、删除、点赞、评论、关注、改状态，都是对同一条记录的操作，
其他入口的内容与统计数字随之一致（无本地副本、不复制生成）。

功能对齐清单（每个入口都具备）：发布 / 编辑 / 删除、评论及回复评论、点赞、显示评论数与点赞数、
设置标签、选择可见范围、查看详情与全文、关注帖子、按权限控制编辑删除。

## 二、发布入口（集成式）

发布入口统一为一个「＋发布讨论」按钮，点击后弹出 `PostComposer`。字段：
讨论类型（研究问题 / 数据问题 / 方法讨论 / 合作招募 / 自由讨论）、标题、正文、
关联数据集（关键词自动匹配）、关联课题组（上下文带入）、标签、可见范围、附件（发布后可下载）。
不同页面的按钮只改变**默认关联对象**，字段与规则完全一致。

## 三、帖子字段与权限

- `posts.title`（标题，可选）、`post_type`（类型）、`status`（open / resolved / closed）、`created_at`。
- 创建：登录用户均可；正文或标题至少填一项。
- 编辑 / 删除：作者本人或平台总管理员。删除帖子会连带清理其评论、点赞、标签、关注、附件文件。
- 评论 / 回复：能看到该帖（满足可见范围）的登录用户即可；删除评论 = 评论作者 / 帖子作者 / 平台管理员，
  删除一级评论会连带删除其下回复。
- 附件：作者上传，凡能看到该帖的人都可下载（走带鉴权的 `downloadFile`）。

## 四、评论数量统计口径（全站统一）

- 统计口径 = **一级评论 + 其下回复的总数**。
- 已删除的评论已被物理删除，不计入。
- 展示简化：≤99 原样；100–999 显示 `99+`；千级显示 `x千+`；万级显示 `x万+`。
- 所有出现帖子卡片的地方（研究广场 / 数据集讨论区 / 课题组 / 我的讨论 / 热榜）都实时显示评论数与点赞数。

## 五、研究热榜（Feed 左栏）

- 热度综合：`点赞 + 3×评论 + 2×独立参与评论人数`，再乘时间衰减（约 72 小时半衰期），
  避免只用历史累计点赞。评论权重高于点赞。
- 时间范围：`24小时 / 7天 / 全部`，默认 7 天。
- 榜单展示前 1–10 条，只显示排名、标题与点赞/评论数，点击直达帖子。
- 接口：`GET /posts/hot?range=`；主信息流排序 `GET /posts?sort=hot|new`。

## 六、筛选（Feed 右栏 + URL 同步）

- 维度：按类型、按课题组、按数据集、仅看未解决、仅看合作招募。
- 层级：选中课题组后，数据集列表只显示该组下的数据集；选中数据集会回填其所属课题组。
- 已选条件以 chip 形式显示在信息流顶部，支持单独取消或一键清除。
- 筛选状态写入页面 URL（`?group=&dataset=&type=&status=&sort=`），便于分享与返回保留。
- 后端 `GET /posts` 支持 `dataset_id / group_id / author_id / post_type / status / scope / sort / range`。

## 七、贡献度可见性（配合修复）

- 公开汇总（总分 + 分类次数，如"数据勘误 12 · 核验 8 · 代码贡献 3"）对任何能访问该主页的人可见
  （`GET /users/{uid}` 的 `contribution` 与 `contribution_breakdown`）。
- 明细按访问者权限过滤（`GET /users/{uid}/contributions`）：本人看全部；他人只看有权访问的记录；
  涉及私有数据集/内部内容的明细不逐条展示，但用「另有 N 项非公开贡献」告知，**汇总分不因此消失**。

## 八、消息与未读（详见 `07-消息中心与邮件提醒规则`）

- 互动通知：评论我的帖子、回复我的评论、点赞我的帖子（按帖子合并）、关注我的帖子、
  代码/数据被评论、被拉入工作台、新版本/新代码发布、各类审批待办。
- 悬浮消息入口显示**红色圆形数字角标**（未读数，超过 99 显示 `99+`），不再只是一个红点。
  角标数 = 待我处理（待办）+ 互动未读数。
- 未读游标存 `notification_state.last_read_at`：晚于游标产生的互动消息计未读；
  打开消息中心不自动清零，只有「全部已读」把游标推进到当前时间才清零（跨设备/刷新一致）。
  待办（审批类）在处理前始终保留。
- 点赞短时间内合并为一条（"X 等 N 人赞了…"）；评论、回复、审批不合并。
- 删除帖子后，其相关评论/点赞/关注一并清理，对应通知自然失效。
